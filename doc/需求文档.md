## 一、产品概述
### 1.1 产品定位
**大湾区经济社会发展大数据智能决策系统** 是一个面向政府部门、研究机构和企业战略部门的数据驱动决策支持平台，通过对大湾区11个城市26年（1999-2024）经济社会数据的多维度分析和智能建模，提供可视化洞察、趋势预测和决策建议。
### 1.2 核心价值
| 价值维度 | 具体体现 |
|---------|---------|
| **数据整合** | 统一11个城市8大指标的历史数据，形成完整的时间序列数据库 |
| **多维洞察** | 支持城市对比、指标关联、趋势分析等多种分析视角 |
| **智能预测** | 基于历史数据的AI预测模型，辅助中长期规划决策 |
| **可视化表达** | 丰富的图表类型和交互方式，降低数据理解门槛 |
## 二、用户画像与场景分析
### 2.1 核心用户画像
| 用户类型 | 角色描述 | 关键需求 | 使用频率 |
|---------|---------|---------|---------|
| **政府决策者** | 发改委、统计局等政府部门负责人 | 快速了解区域发展态势，获取政策制定依据 | 高频/每日 |
| **政策研究员** | 高校、智库研究机构人员 | 深入数据分析，撰写研究报告 | 中频/每周 |
| **企业管理者** | 跨区域企业战略部门 | 城市投资选址、市场进入决策 | 低频/季度 |
| **公众用户** | 大湾区居民、投资者 | 了解区域发展动态 | 低频/偶尔 |
### 2.2 典型使用场景
**场景1：政策制定者制定年度规划**
> 用户需要了解大湾区各城市上一年度GDP增长率、人口变化趋势，以制定新一年的区域协调发展政策。
**场景2：企业选择投资城市**
> 企业希望对比深圳、东莞、佛山三个城市的进出口总额和零售业销售额，选择投资目的地。
**场景3：研究人员撰写区域发展报告**
> 研究员需要分析大湾区26年来各城市产业转移轨迹，通过进出口数据和GDP变化论证观点。

### 3.2 核心表结构DDL
```sql
-- 城市表
CREATE TABLE cities (
    city_id SERIAL PRIMARY KEY,
    city_name VARCHAR(50) NOT NULL UNIQUE,
    city_code VARCHAR(10) NOT NULL,
    city_type VARCHAR(20) NOT NULL, -- 'mainland' or 'hk_macau'
    region VARCHAR(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
-- 指标表
CREATE TABLE indicators (
    indicator_id SERIAL PRIMARY KEY,
    indicator_name VARCHAR(100) NOT NULL UNIQUE,
    indicator_code VARCHAR(20) NOT NULL,
    unit VARCHAR(20),
    category VARCHAR(50),
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
-- 年份数据表（主数据表）
CREATE TABLE annual_data (
    data_id BIGSERIAL PRIMARY KEY,
    city_id INTEGER REFERENCES cities(city_id),
    indicator_id INTEGER REFERENCES indicators(indicator_id),
    year INTEGER NOT NULL,
    value DECIMAL(20,4),
    data_quality VARCHAR(20) DEFAULT 'normal',
    data_source VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(city_id, indicator_id, year)
);
-- 创建索引优化查询
CREATE INDEX idx_annual_data_city ON annual_data(city_id);
CREATE INDEX idx_annual_data_indicator ON annual_data(indicator_id);
CREATE INDEX idx_annual_data_year ON annual_data(year);
CREATE INDEX idx_annual_data_city_indicator ON annual_data(city_id, indicator_id);
-- 预测模型表
CREATE TABLE prediction_models (
    model_id SERIAL PRIMARY KEY,
    model_name VARCHAR(100) NOT NULL,
    model_type VARCHAR(50) NOT NULL,
    model_params JSONB,
    training_start_year INTEGER,
    training_end_year INTEGER,
    accuracy_score DECIMAL(10,6),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
-- 预测结果表
CREATE TABLE predictions (
    prediction_id BIGSERIAL PRIMARY KEY,
    model_id INTEGER REFERENCES prediction_models(model_id),
    city_id INTEGER REFERENCES cities(city_id),
    indicator_id INTEGER REFERENCES indicators(indicator_id),
    year INTEGER NOT NULL,
    predicted_value DECIMAL(20,4),
    confidence_lower DECIMAL(20,4),
    confidence_upper DECIMAL(20,4),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 一、设计原则
### 1.1 核心设计原则
| 原则 | 说明 | 具体应用 |
|-----|------|---------|
| **数据优先** | 突出数据本身，减少装饰干扰 | 图表占主要空间，工具栏次之 |
| **渐进披露** | 根据用户需求逐步展示细节 | 默认展示概览，点击展开详情 |
| **一致性** | 保持视觉和交互的一致性 | 统一的配色、字体、交互模式 |
| **响应性** | 快响应用户操作 | 加载状态反馈，即时数据更新 |
| **可访问性** | 确保所有用户可使用 | 键盘导航，屏幕阅读器支持 |
